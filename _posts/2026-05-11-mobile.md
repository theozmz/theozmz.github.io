## Mobile Operating Systems

* Android OS (Google Inc.)
  * A software platform and operating system for mobile devices.
  * Based on the Linux kernel.
    * Android uses Linux for its device drivers, memory management, process management, and networking.
    * The next level up contains the Android native libraries. They are all written in C / C++ internally, but you’ll be calling them through Java / Kotlin interfaces. In this layer you can find the Surface Manager, 2D and 3D graphics, media codecs, the SQL database (SQLite), and a native web browser engine (WebKit).
  * Developed by Google and later the Open Handset Alliance (OHA).
  * Dalvik Virtual Machine & Android Runtime (ART) Dalvik 虚拟机 & Android 运行时 (ART)
    * Managed runtime used by applications and some system services on Android. Android 上供应用程序和某些系统服务使用的托管运行时。
    * Android 4.4 (KitKat) brought a technology preview of ART, including it as an alternative runtime environment and keeping Dalvik as the default virtual machine. In the subsequent major Android release, Android 5.0 (Lollipop), Dalvik was entirely replaced by ART. Android 4.4 (KitKat) 引入了 ART 的技术预览版，将其作为一种替代运行时环境，并保留 Dalvik 作为默认虚拟机。在随后的主要 Android 版本 Android 5.0 (Lollipop) 中，Dalvik 被 ART 完全取代。
    * Key difference:
      * Dalvik: interpretation and trace-based JIT compilation Dalvik：解释与基于跟踪的即时编译
      * Android Runtime (ART): Android运行时（ART）：
        * ahead-of-time (AOT) compilation, compiling entire applications into native machine code upon their installation -> improves the overall execution efficiency and reduces power consumption, which results in improved battery autonomy on mobile devices 提前（AOT）编译，在安装时将整个应用程序编译成本地机器代码，可提高整体执行效率并降低功耗，从而提高移动设备的电池续航能力
        * faster execution of applications, improved memory allocation and garbage collection (GC) mechanisms, new applications debugging features, and more accurate high-level profiling of applications 更快的应用程序执行速度、改进的内存分配和垃圾回收（GC）机制、新的应用程序调试功能以及更准确的应用程序高级性能剖析
  * Security: 安全性：
    * Android is a multi-process system, in which each application (and parts of the system) runs in its own process. Most security between applications and the system is enforced at the process level through standard Linux facilities, such as user and group IDs that are assigned to applications. Android 是一个多进程系统，其中每个应用程序（以及系统的一部分）都在自己的进程中运行。应用程序与系统之间的大部分安全都通过标准的 Linux 机制在进程级别强制执行，例如分配给应用程序的用户和组 ID。
    * Additional finer-grained security features are provided through a "permission" mechanism that enforces restrictions on the specific operations that a particular process can perform, and per-URI permissions for granting ad-hoc access to specific pieces of data. 通过“权限”机制提供了额外的更细粒度的安全功能，该机制强制执行对特定进程可以执行的特定操作的限制，以及用于授予对特定数据片段的临时访问的每个 URI 的权限。
  * Programming languages: 编程语言：
    * Java / Kotlin – officially supported
    * C / C++ – as native library that Gradle can package with APK (using build tool CMake), and being called by Java / Kotlin program via Java Native Interface (JNI) C / C++ – 作为 Gradle 可以随 APK 一起打包的原生库（使用构建工具 CMake），并通过 Java Native Interface (JNI) 由 Java / Kotlin 程序调用

### Key Principles of Mobile User Experience Design

1. Principle 1: Remove the clutter, protect your design from overloads 原则1：消除杂乱，保护设计免受过载
2. Principle 2: Intuitive navigation 原则2：直观导航
3. Principle 3: Seamless experience for all type of devices 原则3：所有设备类型的无缝体验
4. Principle 4: Design finger-friendly tap-targets 原则4：设计适合手指操作的点击目标
5. Principle 5: Legible content 原则 5：可读内容
6. Principle 6: Clearly visible interface elements 原则6：清晰可见的界面元素
7. Principle 7: Design controls should be based on hand position 原则7：设计控制应基于手部位置
8. Principle 8: Minimize the need for typing 原则8：最小化输入需求

## Lifecycle

* Inception 初始阶段
* Design 设计
  * Paper-based Design Prototype
    * to communicate ideas: between designers, developers, users and other stakeholders in the first stages of the user-centered design process. 用于在以用户为中心的设计过程的早期阶段，在设计师、开发人员、用户和其他利益相关者之间传达想法。
  * User Experience (UX) Design
    * When creating UX mockups, you may need to consider the interface guidelines for the various platforms that the app will target. The app should "feel at home" on each platform. 在创建用户体验模型时，您可能需要考虑应用程序将要面向的各种平台的界面指南。该应用程序在每个平台上都应具有“归属感”。
* Development 开发
* Stabilization 稳定化
  * Prototype – The app is still in proof-of-concept phase and only core functionality, or specific parts of the application are working. Major bugs are present. 原型 – 该应用程序仍处于概念验证阶段，仅核心功能或应用程序的特定部分可以运行。存在重大错误。
  * Alpha – Core functionality is generally code-complete (built, but not fully tested). Major bugs are still present, outlying functionality may still not be present. 核心功能基本代码已完成（已构建，但未完全测试）。仍存在主要错误，部分功能可能尚未实现。
  * Beta – Most functionality is now complete and has had at least light testing and bug fixing. Major known issues may still be present. 大部分功能现已完成，并经过至少初步测试和错误修复。可能仍存在重大的已知问题。
  * Release Candidate – All functionality is complete and tested. Barring new bugs, the app is a candidate for release to the wild. 所有功能均已完成并经过测试。除非出现新错误，否则该应用程序可作为发布到生产环境的候选版本。
* Deployment 部署
  * Distribution
    * iOS Ad-Hoc Deployment – Ad-hoc deployment is intended primarily for development and testing and allows you to deploy to a limited number of properly provisioned devices. When you deploy to a device via Xcode, it is known as ad-hoc deployment. Ad-hoc 部署主要用于开发和测试，并允许您部署到数量有限的已正确配置的设备上。当您通过 Xcode 部署到设备时，这被称为 ad-hoc 部署。
    * In-House Deployment – In-House deployment is meant for internal distribution of corporate applications that aren’t available publicly via the App Store. 内部部署用于不对外公开通过 App Store 分发的企业应用程序的内部分发。
    * Apple App Store – Apple’s App Store is a globally available online application repository, and it allows developers to market and distribute their apps online with very little effort. 苹果的应用商店是一个全球可用的在线应用程序存储库，它允许开发者以极小的努力在线营销和分发他们的应用程序。
  * Performance Tracking and Enhancement
    * The number of downloads 下载次数
    * Total number of minutes spent by each user 每位用户花费的总分钟数
    * Total revenue collected from the mobile app 从移动应用程序收到的总收入

## Java

### ArrayList

* Define an ArrayList: `ArrayList<Class> <Variable Name> = new ArrayList<Class>();`
* Add elements into ArrayList: `<Variable Name>.add();`
* Get element at index i from ArrayList: `<Variable Name>.get(i);`
* Return index of object o or -1 (not found): `<Variable Name>.indexOf(Object o);`
* Get size of (number of elements in) ArrayList: `<Variable Name>.size();`

### OOP

* Abstraction 抽象
  * The act of representing essential features without including background details 提取本质特征而不包含背景细节的行为
* Encapsulation 封装
  * The act of wrapping up of data and operations into a single unit 将数据和操作封装到单个单元中的行为
* Inheritance 继承
  * The process by which objects of one class acquire the properties of objects of another class 一个类别的对象获得另一个类别对象属性的过程
* Polymorphism 多态性
  * The ability to take more than one form 能够采取多种形式的能力

### static

* Java methods and variables can be declared static
* Static variables are independent of any object
* This means that a Class’s
  * static methods can be called even if no objects of that class have been created and
  * static data is “shared” by all instances (i.e., one value per class instead of one per instance)


## XML

### Markup Languages

* Standard Generalized Markup Language (SGML) is a rich meta language that is useful for defining markup languages.
* HTML and XML are based on SGML.
  * Differences: Tags mean something specific to the browser in HTML, they are used for display. Tags mean whatever the user wants them to mean in XML, they are used to describe the data.
* HTML is particularly useful for displaying Web pages (i.e., how to display the data).
* XML defines data structures for electronic commerce (i.e., what is the data).
